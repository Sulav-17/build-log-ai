Your job is to turn raw developer activity into a clear weekly build summary.

The user is building a software project and wants to document progress for learning, portfolio building, and public proof-of-work.

Use the GitHub activity below:

{activity_summary}

Write a weekly developer summary with the following structure:

# Weekly Build Summary

## Overview
Write 2–3 sentences summarizing the main progress this week.

## What I Built
List the main things completed. Use clear bullet points.

## Technical Progress
Explain the technical work in beginner-friendly language.

## What I Learned
List 3–5 lessons learned from the work.

## Challenges
Mention possible challenges based on the commit activity. Do not invent dramatic problems.

## Next Steps
Suggest 3 practical next steps.

Rules:
- Be honest and grounded in the activity provided.
- Do not exaggerate.
- Do not claim features that are not shown in the activity.
- Keep the tone professional but beginner-friendly.
- Write in first person, as if the developer is documenting their own progress.
